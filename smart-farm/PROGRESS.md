## สิ่งที่แก้ในงวดล่าสุด (water level debugging — 2026-06-17)

ปัญหาเดิม: **ค่าระดับน้ำไม่เข้า**. ไล่แก้จนครบ 7 ตัว:

1. **เดิมอ่านแค่เซ็นเซอร์ตัวเดียว** → วนอ่านครบ 7
2. **เข้าใจ protocol ผิด** — SR04M-2 (ป้ายขา TX/RX) จริงๆ ทำงานแบบ **trigger/echo เหมือน HC-SR04** ไม่ใช่ UART. RX=รับ trigger 10µs, TX=ส่ง echo pulse, ระยะ(cm)=µs/58
3. **ยิง trigger ครั้งเดียว อ่าน echo 7 ขาพร้อมกัน** (`triggerAndReadAll`/`measureAllDistances`) — เร็ว + ลด crosstalk
4. **I2C guard** (`bh1750Ok`/`ina219Ok`) — ถ้า setup ไม่เจอ BH1750/INA219 จะ**ข้ามการอ่าน** ไม่งั้น `Wire` ค้างทั้ง loop (เคยทำให้น้ำไม่อ่านเลย)
5. **ฟิลเตอร์ crosstalk** — ทิ้งค่า dist > ความสูงถัง +10cm (เสียงตัวอื่น)
6. **hold ค่าล่าสุด** (`static lastWl[7]`) — กันค่ากระพริบเป็น -1

ตอนนี้: **7 ตัวอ่านได้ครบ, บอร์ดนิ่ง, ส่ง server ได้ (`ok:true`)**

## บทเรียน/กับดักสำคัญ (อย่าหลงซ้ำ)

- **SR04M-2 = trigger/echo ไม่ใช่ UART** (เสียเวลาไล่ UART หลายชั่วโมง). ถ้าอ่าน raw logic เป็น LOW นิ่ง = สายหลวม/ผิดขา ไม่ใช่โค้ด
- **ชื่อขา ESP32:** GPIO36 = **VP**, GPIO39 = **VN** (ไม่พิมพ์เลข). EN = reset ห้ามต่อ
- **I2C guard เช็คแค่ตอน setup** — ต่อ BH1750/INA219 กลับต้อง **กด EN รีบูต** ถึงจะตรวจเจอ
- **BH1750/INA219 ต่อผิด → ลองสลับ SDA(21)/SCL(22)** (เคยแก้ได้ด้วยการสลับ)
- **อย่าขนาน PSU 5V 2 ตัว** — แรงดันสู้กัน เกิด ground offset กวน I2C จนค้าง. PSU 5V 5A ตัวเดียวพอ (เซ็นเซอร์กินรวม <1A)
- **จ่ายไฟเซ็นเซอร์จาก PSU แยก** (ไม่ดึงจากขา ESP32) + **common ground จุดเดียว** — ไม่งั้น ESP32 brownout reboot (`RTCWDT_RTC_RESET`)
- **crosstalk** (ค่าเด้งเมื่อหลายตัวยิงพร้อมกัน) แก้ด้วย **แยกถังจริงมีผนังกั้น** — firmware filter ช่วยได้บางส่วน
- **upload ติด "No serial data received"** → ถอดสาย USB เสียบใหม่ + กด BOOT(ค้าง)+EN
